Important Notes

  • Educational Use Only: This application is intended solely for research and educational purposes. It analyzes previously captured PCAP files and is not designed for real-time monitoring or deployment on live medical systems.
  • Privacy & Legal Notice: Please avoid uploading PCAP files containing real patient data, personal identifiers, or confidential information. Processing PHI (Protected Health Information) without authorization may violate privacy laws.
  • HL7 Traffic Focus: The analyzer detects anomalies in unencrypted HL7 medical network traffic (segments such as MSH|, PID|, OBR|). It identifies patterns that may indicate replay, data exfiltration, or traffic spikes.
  • Encrypted Traffic: Encrypted packets are intentionally excluded from analysis, as decryption would require access to private keys or interception methods outside the ethical and technical scope of this project.
  • Unencrypted HL7 Alerts: These alerts appear whenever plaintext HL7-like content is detected without TLS/SSL encryption. This does not indicate an error â€” it simply confirms that readable HL7 traffic was found. Because the analyzer ignores encrypted packets, this alert will almost always trigger when processing real HL7 data.
  • Detection Method: Pattern-based detection (keyword and segment matching) is used instead of full HL7 protocol parsing. This design keeps the tool lightweight but may produce false positives when non-HL7 traffic contains similar text patterns.
  • Limitations & Future Work: DICOM (medical imaging) traffic is not analyzed in the current version. Future updates aim to add optional parsing for DICOM-based systems (e.g., PACS servers) to detect insecure image transfers.
